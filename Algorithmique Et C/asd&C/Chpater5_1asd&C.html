Pour maîtriser la complexité d’un problème, il est préférable de procéder à une conception modulaire qui consiste à le décomposer en plusieurs sous-problèmes. Ces sous-problèmes peuvent à leurs tours être décomposés jusqu’à aboutir à des traitements élémentaires simples. Ensuite, chaque sous-problème sera résolu par un sous-programme qui peut être une procédure ou une fonction. A titre d’exemple, un programme de gestion de scolarité peut être découpé en plusieurs modules : inscription, suivi des absences, examens, diplômes, etc. (figure 13).

Les modules développés peuvent être réutilisés plusieurs fois dans le même programme ou dans d’autres programmes une fois intégrés à des bibliothèques.

Définition d’une fonction

Une fonction exécute des actions et renvoie un résultat. C'est un morceau de code qui sert à faire quelque chose de précis. On dit qu'une fonction possède une entrée et une sortie. La fig. suivante représente une fonction schématiquement

Lorsqu'on appelle une fonction, il y a trois étapes.

  1. L'entrée: on fait « rentrer » des informations dans la fonction (en lui donnant des informations avec lesquelles travailler).
  2. Les calculs : grâce aux informations qu'elle a reçues en entrée, la fonction travaille.
  3. La sortie : une fois qu'elle a fini ses calculs, la fonction renvoie un résultat. C'est ce qu'on appelle la sortie, ou encore le retour.

Exemple

Pa exemple, une fonction appelée triple qui calcule le triple du nombre qu'on lui donne, en le multipliant par 3 (fig. suivante). Bien entendu, les fonctions seront en général plus compliquées.

Déclaration algorithmique


FONCTION Nom_fonction ( Pf1 : Type1 , … , Pfn : Typen ) : Type_Résultat 
Variables
    Déclarations locales
DEBUT
    Instruction1
    Instruction2
    ……
     InstructionN 
Retourner Résultat
FIN

Déclaration en C


Type_Résultat NomFonction (Type1 Pf1, Type2 Pf2,… , Typen)
{
    Déclarations locales
    instructions
    return Résultat
}
  • Type de résultat (correspond à la sortie) : c'est le type de la fonction. Ce type dépend du résultat que la fonction renvoie. On met un des types que l'on connaît (entier, caractère, réel, etc.) ;
  • nomFonction : c'est le nom de la fonction. (en respectant les mêmes règles que les noms des variables pas d'accents, pas d'espaces, etc.).
  • paramètres ou arguments (correspond à l'entrée) : Ce sont des valeurs avec lesquelles la fonction va travailler. Il y a donc deux sortes de fonctions :
    • Fonction sans passage d’arguments : aucun paramètre envoyer à la fonction, mais ça se fait plus rarement.
    • Fonction avec passage d’arguments : un nombre de paramètres est nécessaire pour la fonction

Fonction avec passage d’arguments (paramètres)

L’entête d’une fonction paramétrée comporte, en plus du nom de la fonction, une liste de paramètres formels (fictifs) séparées par une virgule.

Exemple1


En algorithmique
Fonction triple(nombre : Entier) :Entier
Variables
    resultat : Entier
Debut
    resultat<--3 * nombre // On multiplie le nombre fourni par 3
retourner resultat// On retourne la variable resultat qui vaut le triple de nombre
Fin
    

En C
int triple(int nombre)
{
int resultat ;
resultat = 3 * nombre; // On multiplie le nombre fourni par 3
return resultat; // On retourne la variable resultat qui vaut le triple de nombre
}
  • La fonction triple reçoit une variable de type entier appelée nombre.
  • La fonction est de type entier. Elle doit donc renvoyer une valeur de type entier :La ligne qui donne pour consigne de « renvoyer une valeur » est celle qui contient le « retourrner ». Cette ligne se trouve généralement à la fin de la fonction, après les calculs.

Exemple2

La fonction suivante nommé addition permet d’additionner deux nombres a et b :


En algorithmique
Fonction addition(a : Entier, b :Entier) : Entier
Debut
    retourner a + b;
Fin


En C
int addition(int a, int b)
{
    return a + b;
}

Fonction sans passage d’arguments

Certaines fonctions, ne prennent aucun paramètre en entrée

Exemple1


En algorithmique
Fonction Taille() : Entier
Variables
	L : Entier
Debut
	Répéter
	Ecrire ("Donner une valeur entière comprise entre 10 et 100")
		lire(L)
	Jusqu’à(L >=10 ET L<=100) 
	retourner L
Fin


En C
int Taille ()
{
	int L;
	 do
	{
		printf("Donner une valeur entière comprise entre 10 et 100");
		scanf("%d",&L);
	}
	while (L< 10 || L>100);
	return L;
}

Structure de sélection simple

Forme simple : L'instruction Si ... Alors ...

L'instruction si est la structure de test la plus basique, on la retrouve dans tous les langages (Avec une syntaxe différente...). Elle permet d'exécuter une série d'instructions si jamais une condition est réalisée.

Syntaxe algorithmique :

si (expression vraie) alors
{BLOC D'INSTRUCTIONS}

L’exécution de cette instruction se déroule selon l’organigramme suivant :

Exemple :

si (age >= 18) alors  
ecrire ("Vous etes majeur !")
Fin si

Syntaxe en C :

if (condition)
{
......; /* bloc d'instructions */
}

Exemple :

if (age >= 18) 
{
printf ("Vous etes majeur !");
}

• La condition doit être entre des parenthèses.
• En programmation C, s’il n'y a qu'une instruction, les accolades ne sont pas nécessaires...
• Il est possible de définir plusieurs conditions à remplir avec les opérateurs ET et OU (eq en C : && et ||).

Remarque

• L'instruction suivante teste si les deux conditions sont vraies : 

Syntaxe algorithmique:
si((condition1)ET(condition2))

Syntaxe en C:
if ((condition1)&&(condition2))

• L'instruction suivante exécutera les instructions si l'une ou l'autre des deux conditions est vraie : 

Syntaxe algorithmique:
si((condition1)OU(condition2))

Syntaxe en C:
if ((condition1)||(condition2))

Voici un petit tableau de symboles du langage C à connaître :

Test Signification
if (a==b) test d’égalité
if (a != b) test de non égalité si a est différent de b
if (a > b) Si a est supérieur à b
if (a < b) Si a est inférieur à b
if (a >= b) Si a est supérieur ou égal à b
if (a <= b) Si a est inférieur ou égal à b
if ((expression1)||(expression2)) test de OU LOGIQUE: si l’expression1 ou l’expression2 est vraie
if ((expression1)&&(expression2)) test de ET LOGIQUE : si l’expression1 et l’expression2 sont vraies
if (!(expression1)) test de NON LOGIQUE : si l’expression1 est fausse

Toutes les combinaisons sont possibles entre ces tests.

Forme alternative : L’instruction Si ...Alors….Sinon ...

L'instruction Si dans sa forme basique ne permet de tester qu'un choix, or la plupart du temps on aimerait pouvoir choisir les instructions à exécuter en cas de non réalisation de la condition...

Il s'agit de l'instruction:

Syntaxe algorithmique: 

Si <Condition> Alors

<Séquence d’instructions 1>

Sinon

<Séquence d’instructions 2>

FinSi


Cette expression permet d'exécuter une autre série d'instructions en cas de non-réalisation de la condition.

L’exécution de cette instruction se déroule selon l’organigramme suivant:



Exemple:

Si (age >= 18) alors 

     ecrire("Vous etes majeur !")

Sinon

     printf ("Ah c'est bete, vous etes mineur !");

FinSi


Syntaxe en C:

if (condition réalisée) {

    //liste d'instructions

}

else {

    //autre série d'instructions

}


Exemple:

if (age >= 18) {

     printf ("Vous etes majeur !");

} else {

     printf ("Ah c'est bete, vous etes mineur !");

}
